# Purpose

This document defines the standard lifecycle for all website projects at
Food Sex Alcohol.

Its purpose is to ensure every website project follows a consistent,
repeatable process from authorization through production release and
project closure.

This standard applies to all Website Department projects unless the
Executive Office explicitly approves an alternative workflow.

------------------------------------------------------------------------

# Standard Lifecycle

Every website project follows the same sequence.

Executive Authorization

↓

Website Requirements

↓

Technology Implementation

↓

Website Acceptance

↓

Supporting Department Validation (when applicable)

↓

Executive Release Review

↓

Technology Production Release

↓

Production Verification

↓

Project Closure

No stage may be skipped without Executive approval.

------------------------------------------------------------------------

# Phase 1 --- Executive Authorization

The Executive Office:

-   Defines project scope.
-   Assigns project identifiers.
-   Assigns departmental ownership.
-   Approves project objectives.
-   Defines project constraints.

Deliverable: Project Authorization.

------------------------------------------------------------------------

# Phase 2 --- Website Requirements

The Website Department:

-   Defines reader experience.
-   Defines navigation behavior.
-   Defines acceptance criteria.
-   Defines design constraints.
-   Defines expected deliverables.

The Website Department does not implement code.

Deliverable: Website Requirements.

------------------------------------------------------------------------

# Phase 3 --- Technology Implementation

The Technology Department:

-   Reviews approved requirements.
-   Defines implementation approach.
-   Identifies technical risks.
-   Directs engineering execution through Codex.
-   Validates technical implementation.

Codex performs:

-   Source code changes
-   Commits
-   Pull requests
-   Preview deployments

Deliverable: Technology Implementation Report.

------------------------------------------------------------------------

# Phase 4 --- Website Acceptance

The Website Department independently reviews the implementation.

Acceptance classifications:

-   Accept
-   Accept with Conditions
-   Reject

Acceptance confirms implementation conforms to the approved reader
experience.

Acceptance does not authorize production release.

Deliverable: Website Acceptance Report.

------------------------------------------------------------------------

# Phase 5 --- Supporting Department Validation

Supporting departments participate only when authorized.

Examples include:

-   SEO
-   Analytics
-   Editorial

Each department validates only within its approved authority.

Supporting departments may not expand project scope.

Deliverable: Department Validation Report.

------------------------------------------------------------------------

# Phase 6 --- Executive Release Review

The Executive Office determines:

-   Whether approved work is complete.
-   Whether scope remained controlled.
-   Whether remaining issues belong within the active project.
-   Whether production release is authorized.

Possible outcomes:

-   Approved
-   Approved with Conditions
-   Returned for Remediation

Deliverable: Executive Release Decision.

------------------------------------------------------------------------

# Phase 7 --- Technology Production Release

After Executive approval, Technology directs Codex to:

-   Merge the approved pull request.
-   Deploy through Vercel.
-   Verify production.
-   Confirm successful deployment.

Deliverable: Production Verification Report.

------------------------------------------------------------------------

# Phase 8 --- Project Closure

After successful production verification:

The Executive Office records:

-   Project Closed
-   Final decision reference
-   Production release confirmation

The project becomes part of the permanent company record.

------------------------------------------------------------------------

# Scope Control

Website projects shall not:

-   Expand beyond approved scope.
-   Introduce unrelated improvements.
-   Redesign previously accepted work.
-   Assign work to other departments without Executive approval.

Additional findings shall be classified as:

-   Immediate Remediation
-   Future Improvement
-   Accepted Risk

Executive determines final disposition.

------------------------------------------------------------------------

# Guiding Principles

Every website project shall:

-   Protect the reader experience.
-   Preserve editorial quality.
-   Prefer the smallest effective implementation.
-   Require evidence before acceptance.
-   Separate implementation from governance.
-   Separate acceptance from deployment.
-   Preserve production stability.

------------------------------------------------------------------------

# Success Standard

A website project succeeds when:

-   Approved scope is completed.
-   Reader experience conforms.
-   Supporting departments validate their assigned responsibilities.
-   Executive approves release.
-   Production deployment succeeds.
-   Production verification passes.
-   The project is formally closed.
