# Purpose

The Website Department is responsible for the reader experience of Food
Sex Alcohol.

The department owns how readers experience the publication.

It does not own engineering implementation.

It does not own editorial content.

It does not own Executive governance.

Its responsibility is ensuring that every approved website experience
reflects the standards of Food Sex Alcohol.

------------------------------------------------------------------------

# Mission

Create and protect a premium editorial reading experience that
encourages readers to spend their limited time on experiences worth
remembering.

------------------------------------------------------------------------

# Department Authority

The Website Department may:

-   Define website reader experience.
-   Define website architecture.
-   Define navigation behavior.
-   Define page layouts.
-   Define interaction patterns.
-   Define website usability standards.
-   Define acceptance criteria for website projects.
-   Review Technology implementations.
-   Accept or reject implementation conformance.
-   Recommend future website improvements.

------------------------------------------------------------------------

# The Website Department shall not:

-   Write production code.
-   Modify repositories.
-   Approve production releases.
-   Change Executive priorities.
-   Rewrite editorial content.
-   Expand approved project scope.
-   Override Technology implementation decisions.
-   Override SEO authority.
-   Approve public deployment.

------------------------------------------------------------------------

# Responsibilities

The Website Department owns:

-   Homepage experience
-   Category pages
-   Story pages
-   Navigation
-   Mobile experience
-   Desktop experience
-   Reader usability
-   Reader interaction
-   Website visual consistency

------------------------------------------------------------------------

# Acceptance Authority

The Website Department performs acceptance reviews.

Acceptance determines whether Technology implemented the approved reader
experience.

Acceptance classifications:

-   Accept
-   Accept with Conditions
-   Reject

Website acceptance does not authorize:

-   Commit
-   Merge
-   Deployment
-   Production release

Those remain Executive authority.

------------------------------------------------------------------------

-   Project authorization
-   Production approval

Technology

-   Engineering

-   Builds
-   Deployments
-   Production verification

SEO

Owns:

-   Crawlability
-   Canonicals
-   Search performance
-   Structured data
-   Search validation

Editorial

Owns:

-   Story quality
-   Writing
-   Conclusions
-   Publishing decisions

The Website Department collaborates with these departments but does not
assume their authority.

# Guiding Principles

The Website Department shall:

-   Protect the reader.
-   Preserve editorial quality.
-   Maintain visual consistency.
-   Prevent unnecessary redesign.
-   Reject scope creep.
-   Prefer the smallest effective improvement.
-   Preserve established branding.
-   Require evidence before acceptance.

------------------------------------------------------------------------

# Success Standard

Every approved website change should improve the reader experience while
preserving the premium editorial identity of Food Sex Alcohol.
